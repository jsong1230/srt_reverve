# CLI 워크플로우 (프로세스 흐름)

## 1. 전체 프로세스 흐름

```
+-------------------+
|  사용자 명령행    |
|  quickstart.py    |
|  또는             |
|  manual_login.py  |
+--------+----------+
         |
         v
+---------------------------------+
| Step 1: 설정 로드 (F-07, M1)   |
| - CLI 인자 파싱 (util.py)      |
| - .env 환경변수 폴백            |
| - 기본값 적용                   |
| - Config 객체 생성              |
+--------+------------------------+
         |
         v
+---------------------------------+
| Step 2: 입력 검증 (F-01~F-06)  |
| - 역명 검증 (station_list)     |

| - 날짜 유효성 (datetime 파싱)  |

| - 실패 시 즉시 종료 (exit 2)   |
+--------+------------------------+
         |
         v
+----------------------------------+

| - Chrome 옵션 설정               |
| - 봇 탐지 우회 방식 선택         |
|   (undetected/stealth/enhanced)  |
| - 인간 행동 시뮬레이션 주입      |
| - 드라이버 생성                  |
+--------+-------------------------+
         |
         v
+----------------------------------+
| Step 4: 로그인 (F-01)            |
| - SRT 로그인 페이지 접속         |
| - ID/비밀번호 입력               |
|   (인간 타이핑 시뮬레이션)       |
| - 로그인 버튼 클릭               |
| - check_login() 확인             |
| - 실패 시 F-08 에러 복구 (M1)   |
+--------+-------------------------+
         |
         v
+----------------------------------+
| Step 5: 기차 검색 (F-02)         |
| - 검색 페이지 이동               |
| - 출발역/도착역 입력             |
| - 날짜/시간 선택                 |
| - 조회 버튼 클릭                 |
| - 검색 결과 표시 대기            |
+--------+-------------------------+
         |
         v
+--------------------------------------+
| Step 6: 예약 반복 (F-03, F-11 M3)   |
| +----------------------------------+ |
| | 루프 시작 (무한 반복)             | |
| +----------------------------------+ |
| | 1. 상위 N개 기차 결과 확인       | |
| | 2. "예약하기" 발견?              | |
| |    -> YES: book_ticket() 시도    | |
| |    -> 성공: Step 7로 이동        | |
| | 3. "신청하기" 발견? (F-04)       | |
| |    -> want_reserve=True이면      | |
| |       reserve_ticket() 시도      | |
| |    -> 성공: Step 7로 이동        | |
| | 4. 실패: {delay}초 랜덤 대기     | |
| |    (60~120초, 봇 탐지 회피)      | |
| | 5. 새로고침 -> 루프 반복         | |
| | 6. 에러 발생 시 (M1):            | |
| |    -> 네트워크: 자동 재시도      | |
| |    -> 세션 만료: 재로그인        | |
| |    -> 크래시: WebDriver 재초기화 | |
| +----------------------------------+ |
+--------+-----------------------------+
         |
         v
+----------------------------------+
| Step 7: 결과 보고                 |
| - 예약 성공 메시지 출력           |
| - Telegram 알림 발송 (F-09, M2)  |
| - 로그 파일 기록 (F-10, M2)      |
+--------+-------------------------+
         |
         v
+----------------------------------+
| Step 8: 종료                      |
```

## 2. 진입점별 차이

### quickstart.py (자동 로그인)

```
CLI 인자 파싱 (--user, --psw 필수)
    -> Config 로드 (M1)
    -> SRT 인스턴스 생성
    -> srt.run(login_id, login_psw)
        -> run_driver() -> login() -> go_search() -> check_result()
```

### manual_login.py (수동 로그인)

```
CLI 인자 파싱 (--user, --psw 불필요)
    -> Config 로드 (M1)
    -> SRT 인스턴스 생성
    -> run_driver()
    -> SRT 로그인 페이지 이동
    -> "로그인 후 Enter를 누르세요" (stdin 대기)
    -> check_login() 확인
    -> go_search() -> check_result()
```

차이점:
- `use_profile = False` (충돌 방지)
- `retry_delay_min/max = 150/300` (더 긴 대기)
- `anti_bot_method = 'stealth'` (수동 모드 기본값)

## 3. 에러 복구 흐름 (F-08, M1)

```
[에러 발생]
    |
    +---> [네트워크 오류?] (TimeoutException, ConnectionError)
    |         |
    |         +---> YES: [재시도 가능?] (retry_count < 3)
    |         |         |
    |         |         +---> YES: [{delay}초 대기] -> [동일 작업 재시도]
    |         |         |                                    |
    |         |         |                              +---> [성공] -> 정상 복귀
    |         |         |                              +---> [실패] -> 재시도 카운트 증가 -> 루프
    |         |         |
    |         |         +---> NO: [최종 실패] -> 에러 메시지 + exit 5
    |         |
    |         +---> NO: 다음 판단으로
    |
    +---> [세션 만료?] (로그인 페이지 리다이렉트 감지)
    |         |
    |         +---> YES: [재로그인 가능?] (session_retry < 2)
    |         |         |
    |         |         +---> YES: [자동 재로그인] -> [검색 재개]
    |         |         +---> NO: [최종 실패] -> 에러 메시지 + exit 3
    |         |
    |         +---> NO: 다음 판단으로
    |
    +---> [브라우저 크래시?] (WebDriverException, 세션 끊김)
    |         |
    |         +---> YES: [복구 가능?] (crash_retry < 1)
    |         |         |
    |         |         +---> YES: [WebDriver 재초기화] -> [로그인부터 재시작]
    |         |         +---> NO: [최종 실패] -> 에러 메시지 + exit 5
    |         |
    |         +---> NO: 다음 판단으로
    |
    +---> [입력 오류?] (InvalidStation/Date/TimeError)
    |         |
    |         +---> [즉시 종료] -> 에러 메시지 + exit 2
    |
    +---> [미분류 오류]
              |
              +---> [로깅 후 종료] -> 에러 메시지 + exit 99
```

## 4. 마일스톤별 워크플로우 확장

### M0 (현재): 기본 흐름

```
CLI 인자 -> SRT 초기화 -> WebDriver -> 로그인 -> 검색 -> 예약 루프 -> 종료
```

변경 없음. 현재 구현된 흐름 그대로.

### M1: 설정 관리 + 에러 복구

Step 1 확장:
```
CLI 인자 + .env 파일 -> Config.from_env_and_cli() -> Config 객체 -> SRT 초기화
```

에러 복구 추가:
```
각 단계에서 에러 발생 시 -> RecoveryManager 판단 -> 자동 복구 또는 종료
```

### M2: 알림 + 로깅

Step 7 확장:
```
예약 결과 -> TelegramNotifier.notify_success/failure() -> Telegram 메시지 발송
```

전체 흐름 확장:
```
모든 단계 -> Logger (콘솔 + 파일 동시 기록)
```

### M3: 다중 조건 + 헤드리스

Step 5 확장 (F-11):
```
Config.dpt_dt_list / dpt_tm_list
    -> 조건 1: (날짜A, 시간A) 검색
    -> 조건 2: (날짜A, 시간B) 검색
    -> 조건 3: (날짜B, 시간A) 검색
    -> ... (모든 조합 순차 검색)
    -> 1회전 완료 -> 처음부터 반복
```

Step 3 확장 (F-12):
```
Config.headless = True
    -> Chrome 옵션에 --headless=new 추가
    -> 예약 결과를 콘솔 로그로만 확인 (브라우저 창 없음)
```

## 5. 상태 전이 다이어그램

### SRT 인스턴스 상태

```
[INIT] --check_input()--> [VALIDATED]
   |                          |
   | (검증 실패)              | run_driver()
   v                          v
[ERROR]                  [DRIVER_READY]
                              |
                              | login()
                              v
                         [LOGGED_IN]
                              |
                              | go_search()
                              v
                         [SEARCHING]
                              |
                         +----+----+
                         |         |
                    (좌석 발견)  (매진)
                         |         |
                         v         v
                    [BOOKING]  [REFRESHING] --{delay}--> [SEARCHING]
                         |
                    +----+----+
                    |         |
               (성공)    (실패)
                    |         |
                    v         v
               [BOOKED]  [SEARCHING]
```

### Config 로드 상태 (F-07, M1)

```
[CLI_ARGS] --parse--> [PARSED]
                          |
                          v
                     [ENV_LOADED] --.env 파일 읽기
                          |
                          v
                     [DEFAULTS_APPLIED]
                          |
                          v
                     [VALIDATED] --실패--> [ERROR]
                          |
                          v
                     [READY]
```

### RecoveryManager 상태 (F-08, M1)

```
[IDLE] --에러 감지--> [DIAGNOSING]
                          |
                     +----+----+----+
                     |         |    |
                (네트워크) (세션) (크래시)
                     |         |    |
                     v         v    v
               [RETRYING] [RELOGIN] [RESTARTING]
                     |         |    |
                +----+    +----+    +----+
                |    |    |    |    |    |
           (성공)(실패)(성공)(실패)(성공)(실패)
                |    |    |    |    |    |
                v    v    v    v    v    v
           [IDLE] [FAIL] [IDLE] [FAIL] [IDLE] [FAIL]
```

## 7. 사용자 인터랙션 포인트

CLI 애플리케이션이므로 사용자 인터랙션은 최소화되어 있다.

| 시점 | 인터랙션 | 진입점 |
|------|----------|--------|
| 프로그램 시작 | CLI 인자 입력 | 모든 진입점 |
| 수동 로그인 대기 | stdin Enter 입력 | manual_login.py만 |
| 예약 완료 후 | Chrome 창에서 결과 확인 (수동) | quickstart.py, manual_login.py |
| 예약 완료 후 | Chrome 창 수동 닫기 | 모든 진입점 (detach 모드) |
