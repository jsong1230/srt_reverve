# CLI 출력 스타일 가이드

## 1. 개요

SRT 자동 예약 프로그램의 터미널 출력 스타일을 표준화한다. CLI 애플리케이션이므로 UI 디자인 시스템 대신 로그 포맷, 메시지 템플릿, 에러 메시지 체계를 정의한다.

## 2. 로그 포맷 표준

### 기본 포맷 (F-10 구현 전, 현재)

```
logging.basicConfig 기본 포맷
```

### 표준 포맷 (F-10 구현 후, M2)

```
[{YYYY-MM-DD HH:MM:SS}] [{LEVEL}] {message}
```

예시:
```
[2026-03-11 14:30:46] [INFO] SRT 로그인 성공
[2026-03-11 14:30:47] [WARNING] 네트워크 타임아웃 (1/3 재시도)
[2026-03-11 14:30:48] [ERROR] 로그인 실패: 잘못된 비밀번호
```

### 레벨별 용도

| 레벨 | 용도 | 예시 |
|------|------|------|
| DEBUG | 내부 디버그 정보 (CSS Selector, 요소 상태, API 응답 상세) | `[DEBUG] WebDriver 세션 ID: abc123` |
| INFO | 단계별 진행, 정상 상태 변화 | `[INFO] 기차 검색 시작: 동탄 -> 동대구` |
| WARNING | 재시도 가능한 문제, 자동 복구 진행 | `[WARNING] Alert 발생, 자동 처리 완료` |
| ERROR | 재시도 불가능한 치명적 오류 | `[ERROR] 최대 재시도 횟수 초과, 프로세스 종료` |

### 파일 로그 (F-10, M2)

- 파일명: `logs/srt_{YYYY-MM-DD}.log`
- 로테이션: `TimedRotatingFileHandler`, 7일 보관
- 포맷: 콘솔과 동일 (`[{timestamp}] [{level}] {message}`)
- 듀얼 핸들러: 콘솔 + 파일 동시 출력

## 3. 메시지 템플릿

### 3.1 프로세스 시작

```
============================================================
SRT 자동 예약 시작
- 출발: {dpt_stn} -> 도착: {arr_stn}
- 날짜: {dpt_dt}, 시간: {dpt_tm}시 이후
- 봇 탐지 우회: {anti_bot_method}
- 확인 기차 수: {num_trains}개
============================================================
```

### 3.2 단계별 진행 메시지

| 단계 | 메시지 |
|------|--------|
| WebDriver 초기화 | `[INFO] Chrome WebDriver 초기화 중... ({anti_bot_method} 모드)` |
| WebDriver 완료 | `[INFO] Chrome WebDriver 초기화 완료` |
| 로그인 시도 | `[INFO] SRT 로그인 시도` |
| 로그인 성공 | `[INFO] SRT 로그인 성공` |
| 검색 시작 | `[INFO] 기차 검색 시작: {dpt_stn} -> {arr_stn} ({dpt_dt} {dpt_tm}시)` |
| 새로고침 | `[INFO] 검색 결과 새로고침 ({cnt_refresh}회차, {delay}초 대기 후)` |
| 좌석 발견 | `[INFO] 예약 가능 좌석 발견! 예약 시도 중...` |
| 예약 시도 | `[INFO] {i}번째 기차 예약 시도` |

### 3.3 예약 성공

```
============================================================
예약 성공!
- 열차: {출발시간} ~ {도착시간}
- 좌석: {seat_type}
- 출발: {dpt_stn} -> 도착: {arr_stn}
- 새로고침 횟수: {cnt_refresh}회
============================================================
```

### 3.4 예약 대기 신청 성공

```
============================================================
예약 대기 신청 완료!
- 열차: {출발시간}
- 출발: {dpt_stn} -> 도착: {arr_stn}
- 대기 번호는 SRT 앱에서 확인하세요.
============================================================
```

### 3.5 대기 중 상태 (새로고침 주기마다)

```
[INFO] 대기 중... (새로고침 {cnt_refresh}회, 경과 {elapsed})
```

### 3.6 에러 복구 메시지 (F-08, M1)

```
[WARNING] 네트워크 오류 발생, 자동 재시도 ({current}/{max})
[WARNING] 세션 만료 감지, 자동 재로그인 시도
[WARNING] 브라우저 크래시 감지, WebDriver 재초기화 중...
[INFO] 복구 성공, 검색 재개
[ERROR] 최대 재시도 횟수 초과, 프로세스 종료
```

### 3.7 Telegram 알림 메시지 (F-09, M2)

성공 메시지:
```
[SRT 예약 성공]
열차: {dpt_stn} -> {arr_stn}
날짜: {dpt_dt} {dpt_tm}시
좌석: {seat_type}
새로고침: {cnt_refresh}회
```

실패 메시지:
```
[SRT 예약 실패]
원인: {error_msg}
재시도: {retry_count}회
경과 시간: {elapsed}
```

## 4. 에러 메시지 체계

### 4.1 입력 검증 에러 (즉시 종료)

| 예외 클래스 | 메시지 패턴 | 예시 |
|-------------|-------------|------|
| `InvalidStationNameError` | `{역명}은(는) 유효하지 않은 역명입니다. 가능한 역: {역 목록}` | `강남은(는) 유효하지 않은 역명입니다. 가능한 역: 수서, 동탄, ...` |
| `InvalidDateFormatError` | `날짜 형식이 올바르지 않습니다. YYYYMMDD 형식으로 입력하세요: {입력값}` | `날짜 형식이 올바르지 않습니다. YYYYMMDD 형식으로 입력하세요: 2026031` |
| `InvalidDateError` | `유효하지 않은 날짜입니다: {입력값}` | `유효하지 않은 날짜입니다: 20260230` |
| `InvalidTimeFormatError` | `시간은 짝수여야 합니다 (00, 02, ..., 22): {입력값}` | `시간은 짝수여야 합니다 (00, 02, ..., 22): 09` |

### 4.2 필수 인자 누락 에러

```
에러: 필수 인자가 누락되었습니다: {누락 인자 목록}
사용법: python quickstart.py --user USER --psw PASSWORD --dpt 출발역 --arr 도착역 --dt 날짜 --tm 시간
```

### 4.3 런타임 에러

| 상황 | 메시지 |
|------|--------|
| 로그인 실패 | `[ERROR] 로그인 실패: {에러 상세}` |
| 검색 페이지 로드 실패 | `[ERROR] 검색 페이지 이동 실패: {에러 상세}` |
| 예약 클릭 실패 | `[WARNING] 예약 버튼 클릭 실패, JavaScript 클릭으로 재시도` |
| 브라우저 세션 끊김 | `[ERROR] 브라우저 연결이 끊어졌습니다. Chrome 창이 닫혔거나 크래시되었을 수 있습니다.` |

### 4.4 에러 복구 실패 (F-08, M1)

```
[ERROR] 네트워크 오류 복구 실패 (3/3회 재시도 초과)
  마지막 오류: {exception_name}: {exception_message}
  권장 조치: 네트워크 연결 상태를 확인한 후 다시 실행하세요.

[ERROR] 세션 재로그인 실패 (2/2회 재시도 초과)
  마지막 오류: {exception_name}: {exception_message}
  권장 조치: SRT 회원 ID와 비밀번호를 확인한 후 다시 실행하세요.

[ERROR] 브라우저 복구 실패
  마지막 오류: {exception_name}: {exception_message}
  권장 조치: Chrome 브라우저와 ChromeDriver 버전 호환성을 확인하세요.
```

## 5. 종료 코드 (Exit Codes)

| 코드 | 의미 | 원인 |
|------|------|------|
| 0 | 성공 | 예약 또는 대기 신청 완료 |
| 1 | 입력 오류 | 필수 인자 누락, 형식 오류 |
| 2 | 검증 오류 | 역명, 날짜, 시간 유효성 실패 |
| 3 | 로그인 실패 | 최대 재시도 초과 |
| 4 | 검색 실패 | 검색 페이지 로드 실패 |
| 5 | 예약 실패 | 최대 재시도 초과 |
| 99 | 예기치 않은 오류 | 미처리 예외 |

## 6. 출력 규칙

### 일반 규칙

- 구분선(`=====`)은 60자 고정
- 한 줄 최대 80자 (로그 메시지)
- 날짜 포맷: `YYYY-MM-DD`
- 시간 포맷: `HH:MM:SS`
- 한국어 메시지 사용 (사용자 대상 출력)
- 영어 메시지: 기술 로그(DEBUG 레벨)에서만 허용

### 로그 레벨 기본 출력

| 실행 모드 | 콘솔 기본 레벨 | 파일 레벨 (F-10) |
|-----------|---------------|-----------------|
| 일반 실행 | INFO | INFO |
| `--log-level DEBUG` | DEBUG | DEBUG |
| 운영 권장 | INFO | INFO |
| 문제 디버깅 | DEBUG | DEBUG |

### 민감 정보 마스킹

- 비밀번호: 절대 로그에 출력하지 않음
- 회원 ID: 앞 4자리만 표시 (예: `1234****`)
- Telegram Token: 로그에 출력하지 않음
