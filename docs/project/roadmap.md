# 프로젝트 로드맵

## Milestone 0: 프로젝트 기반 구축
- **목표**: 개발 환경 정비, 시스템 분석, 설계 문서 작성
- **작업**: /init-dev (테스트 환경 정비, 린트/포맷 설정) + /design (시스템 분석 문서)
- **완료 기준**:
  - system-analysis.md 작성 완료
  - 기존 테스트 전체 통과
  - 개발 환경 표준화 완료

## Milestone 1: 안정성 강화 (Must)
- **목표**: 기존 기능의 안정성을 높이고 설정 관리를 개선하여 사용 편의성 향상
- **기능**: F-07 (설정 관리), F-08 (에러 리커버리)
- **병렬 그룹 A**: F-07, F-08 (상호 의존성 없음, 동시 개발 가능)
- **외부 의존성**: python-dotenv 패키지
- **완료 기준**:
  - .env 파일로 모든 설정값 관리 가능
  - 네트워크 오류 발생 시 자동 재시도 후 검색 재개
  - 세션 만료 시 자동 재로그인 성공
  - 브라우저 크래시 후 자동 복구 성공
  - 기존 테스트 + 신규 테스트 전체 통과

## Milestone 2: 사용자 경험 개선 (Should)
- **목표**: 알림 기능과 로깅 개선으로 사용자가 프로그램 상태를 쉽게 파악
- **기능**: F-09 (Telegram 알림), F-10 (로깅 개선)
- **병렬 그룹 B**: F-09, F-10 (상호 의존성 없음, 동시 개발 가능)
- **외부 의존성**: Telegram Bot API (python-telegram-bot 또는 requests 직접 호출)
- **완료 기준**:
  - 예약 성공/실패 시 Telegram 메시지 수신
  - Telegram 미설정 시 에러 없이 정상 동작
  - logs/ 디렉토리에 날짜별 로그 파일 생성
  - --log-level 옵션 동작 확인

## Milestone 3: 기능 확장 (Could)
- **목표**: 고급 사용자를 위한 확장 기능 제공
- **기능**: F-11 (다중 검색 조건), F-12 (헤드리스 모드)
- **외부 의존성**: 없음
- **완료 기준**:
  - 여러 날짜/시간 조건으로 순차 검색 후 예약 성공
  - --headless 옵션으로 백그라운드 실행 및 예약 성공
  - 헤드리스 모드에서 봇 탐지 우회 정상 동작

## Milestone 4: 통합 테스트 및 배포
- **목표**: 전체 기능 통합 검증 및 안정 버전 릴리스
- **작업**: /test (전체 회귀 테스트) + /deploy (릴리스 태깅)
- **완료 기준**:
  - 전체 테스트 스위트 통과
  - 기존 기능(F-01~F-06) 회귀 테스트 통과
  - 신규 기능(F-07~F-12) 인수조건 전체 충족
  - README 및 사용 가이드 업데이트 완료
